About Rose

Rose is an imperative programming language which is intended to be scalable (like Scala), nice to read (like Ruby) and intuitive and powerful (like Python). It's also trying to be different, and so it doesn't do the things in traditional way.

Take a look...

Philosophy

  1. Long and nice is better than short and fast but ugly.
  2. Things shouldn't get silently by. Errors should be suppressed only explicitly.
  3. Flat is better than nested.
  4. Size of file is the second thing after readability of long lines (see 1).
  5. No special cases. Just follow the rules. Trust me, they're very benevolent.
  6. There is only one - the nicest, shortest, fastest, somethingest - way to do something.
  7. It's bad idea if you can't explain it easy.

PEP 20 -- The Zen of Python, modified by Jan Růžička

Basic syntax

Syntax is very obvious, it looks like Python. Blocks are indented by 3 whitespaces. And just and only with 3 whatespaces. Functions without parameters need to have empty parenthesis (this will probably change). Rose has macros (expression and block). You can transform operators into functions (like in Ruby, but a bit differently). 

print("Hello world!")  # => "Hello world!"

PI = 3.14159  # PI!!float


# Rose is dynamic

a = 1  # a!!int
a = "Foo!"  # a!!str
a = 'a'  # a!!chr

if a!!bool:  # Type check
  print("`a` is a boolean!")

else:
  print(a.cls.__name__)  # Object has reference to its class. Class has a reference to its name.

print(q["Hello world!"])  # Expression macro - quasiquote, => something like "node.str(v: "Hello world!")"

error:  # Block macro - error. Raises error if the first expression is true.
  PI == 3

  "PI can't be 3!"

assert:  # Same as above. Raises error if the first expression is false.
  PI != 3

  "PI can't be 3!"

def foo(bar, baz=3):  # Function. Last argument is optional/default.
  print(bar * baz)  # This function returns `nil`. The last expression is returned.

  pass  # This "hack" does not work. `pass`es are automatically trimmed. They're used only for parsing.

foo("Hello ")  # => "Hello Hello Hello "
foo(3, 9)  # => "27", function arguments are dynamic. You will be probably able to change this explicitly.

# Have I said that Rose should be different?
# Yes, and the difference is in the object model (or at least in object creation).

def Person(@name, @age):  # Required keyword - non-positional - arguments
  @{name: @name, age: @age}  # Syntatic sugar for object creation.

# So it's just a function returning a dictionary.
